All authenticated endpoints use a single credential — a panel API key — sent in an HTTP header.

## Header

```http
X-Api-Key: sk_live_xxxxxxxxxxxx
```

## Generating a key

You can generate an API key in two ways:

1. From your account dashboard.
2. By calling [`POST /api-keys`](/api-reference/api-keys/create) with an existing key.

<Warning>
  The full `api_key` value is shown **exactly once**, in the response to `POST /api-keys`. Store it immediately in a secret manager — it cannot be retrieved again.
</Warning>

## Rotating a key

If a key leaks, only that one account is affected. Rotate it:

1. Create a new key with [`POST /api-keys`](/api-reference/api-keys/create).
2. Update your integration to use the new key.
3. Revoke the old key with [`DELETE /api-keys/{id}`](/api-reference/api-keys/revoke).

## Access tiers

| Badge | Meaning |
| --- | --- |
| 🌐 **PUBLIC** | No credential required |
| 🔑 **API KEY** | Requires `X-Api-Key` |

Endpoints marked ⚠️ **SENSITIVE** touch payment data, credentials, or a secret revealed exactly once — handle them with extra care (redact in logs, never cache one-time secrets).
