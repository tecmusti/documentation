Image and video generation share the **same** `POST /generate` endpoint. Which validation path runs is decided by the tool the `model` you send belongs to — nothing else in the payload matters for routing.

## Async model

Every successful `POST /generate` returns immediately with `status: "queued"` and a `task_id`.

<Warning>
  You **must** save `task_id` — it's the only handle to poll [`GET /status`](/api-reference/generate/status), and it is never shown again.
</Warning>

Typical completion times:

- **Image:** 10–100 seconds
- **Video:** ~1 minute (noticeably longer than image)

Poll every few seconds — don't long-poll.

## Not implemented

A product-description tool is planned but **not built yet**. Don't send `model` values for it.
