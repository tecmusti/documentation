## `POST /wallet/auto-topup`

### Enable

```json
{
  "enabled": true,
  "payment_method_id": "pm_1TrRXJ4B5rHPrz57h5VULXud",
  "threshold_credits": 5,
  "amount_credits": 1000
}
```

### Disable

```json
{ "enabled": false }
```

<Note>
  Disabling only flips the flag. Previous threshold, amount, and card are retained — a later re-enable can just send `{ "enabled": true }` (though the API currently requires the full set of fields when enabling).
</Note>

### Body parameters

<ParamField body="enabled" type="boolean" required>
  `true` to enable, `false` to disable.
</ParamField>

<ParamField body="payment_method_id" type="string">
  Required when enabling. Must belong to the caller (see [`GET /wallet/payment-methods`](/api-reference/wallet/payment-methods)).
</ParamField>

<ParamField body="threshold_credits" type="integer">
  Required when enabling. Must be greater than 0.
</ParamField>

<ParamField body="amount_credits" type="integer">
  Required when enabling. Minimum **500** credits (\$5.00) — same floor as a manual topup.
</ParamField>

### Response

```json
{ "success": true, "data": { "status": "updated" } }
```

### Errors

| Status | `error.code` | Meaning |
| --- | --- | --- |
| 400 | `invalid_input` | Missing `payment_method_id`, `threshold_credits` not positive, or `amount_credits` below 500. |
| 404 | `not_found` | `payment_method_id` doesn't exist or belongs to another user. |
