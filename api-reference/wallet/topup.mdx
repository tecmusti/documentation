## `POST /wallet/topup`

Starts a Stripe Checkout session. Redirect the user's browser to the returned `checkout_url` to complete payment.

### Request

```json
{
  "credits": 1000,
  "success_url": "https://example.com/wallet?topup=success",
  "cancel_url": "https://example.com/wallet?topup=cancelled",
  "enable_auto_topup": true,
  "auto_topup_threshold": 5,
  "auto_topup_amount": 1500
}
```

### Body parameters

<ParamField body="credits" type="integer" required>
  Number of credits to purchase. 1 credit = $0.01. **Minimum 500** ($5.00). No fixed tiers.
</ParamField>

<ParamField body="success_url" type="string">
  URL to redirect to after successful payment. Defaults to a generic post-checkout page.
</ParamField>

<ParamField body="cancel_url" type="string">
  URL to redirect to if the user cancels checkout. Defaults to a generic post-checkout page.
</ParamField>

<ParamField body="enable_auto_topup" type="boolean">
  If `true`, saves the payment method and enables auto-topup after this payment. Requires `auto_topup_threshold` and `auto_topup_amount`.

  <Warning>
    Enabling here **replaces** any previously-configured auto-topup card.
  </Warning>
</ParamField>

<ParamField body="auto_topup_threshold" type="integer">
  Required when `enable_auto_topup: true`. Must be greater than 0. When the balance drops below this many credits, auto-topup fires.
</ParamField>

<ParamField body="auto_topup_amount" type="integer">
  Required when `enable_auto_topup: true`. Minimum **500** credits (\$5.00). The amount charged on each auto-topup.
</ParamField>

### Optional header

<ParamField header="X-USER-LANG" type="string">
  2-letter ISO language code. Stored against the account and used to localize the topup receipt email. Defaults to English if never sent.
</ParamField>

### Response — 200

```json
{ "success": true, "data": { "checkout_url": "https://checkout.stripe.com/c/pay/cs_test_..." } }
```

Redirect the user's browser to `checkout_url`.

### After payment

<Note>
  Crediting happens automatically and asynchronously once Stripe confirms payment — typically within a few seconds of checkout completing. Poll [`GET /wallet/balance`](/api-reference/wallet/balance) afterward. There is no separate "confirm payment" call.
</Note>

### Errors

| Status | `error.code` | Meaning |
| --- | --- | --- |
| 400 | `invalid_input` | `credits` below 500, or an invalid auto-topup sub-field when `enable_auto_topup: true`. |
